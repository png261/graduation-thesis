# Task Conventions Skill

Use this skill for all non-trivial implementation tasks.

## Conventions

- Use `write_todos` for complex multi-step work.
- Do not enumerate the whole skill library up front; open only the specific skill files needed for the task at hand.
- Do not create `TASKS.md` or other repo-backed planning files.
- Update todos immediately after each completed step and before final completion.
- Keep responses deterministic, concise, and implementation-focused.
